# Random Forest によるサロゲートモデル

## 概要

Random Forest（ランダムフォレスト）は、複数の決定木を組み合わせた**アンサンブル学習**手法。各決定木を異なるブートストラップサンプルで学習し、予測値を平均することで汎化性能を高める。Ridge 回帰と異なり、**非線形・不連続な目的関数**に対しても有効なサロゲートモデルを構築できる。

SurfacePlot3D では、2 パラメータの部分空間に射影した Random Forest を使い、50×50 グリッド上での予測値を応答曲面として可視化する。

---

## CART 決定木

Random Forest の基本構成要素は **CART（Classification and Regression Tree）** と呼ばれる回帰木。

### 木の構造

```
内部ノード (Split): 特徴量 j と閾値 t で左右に分岐
                    x[j] <= t → 左の子ノードへ
                    x[j] >  t → 右の子ノードへ

葉ノード (Leaf):    訓練サンプルの平均値 ȳ を返す
```

Rust では `TreeNode` enum で表現する:

```rust
enum TreeNode {
    Leaf(f64),
    Split { feature: usize, threshold: f64,
            left: Box<TreeNode>, right: Box<TreeNode> },
}
```

### MSE 最小化による分岐探索

各分岐点 `(j, t)` は、分岐後の**加重 MSE（平均二乗誤差）** を最小化するように選択する:

```
Gain(j, t) = MSE(y) − [ n_L/n · MSE(y_L) + n_R/n · MSE(y_R) ]
```

ここで:

- `y_L`, `y_R`：左右に分かれたサンプルの目的値
- `n_L`, `n_R`：左右のサンプル数（`n = n_L + n_R`）

```
MSE(y) = (1/n) Σ_i (y_i − ȳ)²
```

最も Gain が大きい `(j, t)` を最良分岐点として採用する。

### 停止条件

再帰的な木構築は、次のいずれかで停止して葉ノードを生成する:

| 条件                    | 意味                                           |
| ----------------------- | ---------------------------------------------- |
| `depth >= max_depth`    | 最大深さに達した                               |
| `n <= min_samples_leaf` | サンプルが少なすぎる                           |
| 有効な分岐が存在しない  | すべての閾値で `min_samples_leaf` を満たせない |

実装のデフォルト値:

| パラメータ         | 値  | 根拠                                      |
| ------------------ | --- | ----------------------------------------- |
| `max_depth`        | 10  | 過学習を抑制しつつ非線形を捉える深さ      |
| `min_samples_leaf` | 2   | 少サンプルの Study（20〜50 試行）にも対応 |

## Bagging（Bootstrap Aggregating）

### ブートストラップサンプリング

`n_trees` 本の決定木を、それぞれ異なる**復元抽出サンプル**で学習する:

```
訓練データ: {(x_1, y_1), ..., (x_N, y_N)}

木 b の学習データ: 元の N 点からランダムに N 点を復元抽出
  → 平均 63.2% の点が選ばれ、36.8% は重複または未選択（OOB サンプル）
```

Rust 実装では外部クレートを使わず、**LCG（線形合同法）** 疑似乱数を使用:

```rust
// Knuth の定数
state = state * 6364136223846793005 + 1442695040888963407
```

### アンサンブル予測

各木の予測値を平均して最終予測とする:

```
ŷ(x) = (1 / n_trees) Σ_{b=1}^{n_trees} T_b(x)
```

バイアスと分散のトレードオフ:

- **単一決定木**: 低バイアス・高分散 → 過学習しやすい
- **アンサンブル平均**: 分散が `1/n_trees` 倍に減少 → 汎化性能が向上

`n_trees = 100` の場合、分散は単一木の 1/100 になる（各木が独立と仮定）。

## 2D 部分空間射影

### 射影の手順

全パラメータ空間から 2 列を抽出して 2D Random Forest を学習する:

```
元データ: x ∈ R^p  (p パラメータ)
                ↓
2D 射影:  x_{2D} = [x[param1_idx], x[param2_idx]]  ∈ R²
                ↓
Random Forest を R² 上で学習・予測
```

### グリッド予測

```
grid1 = linspace(min(x1), max(x1), n_grid)   # 通常 n_grid = 50
grid2 = linspace(min(x2), max(x2), n_grid)

values[i][j] = RF.predict([grid1[i], grid2[j]])
```

計算量: `n_grid² × n_trees × max_depth = 50² × 100 × 10 = 2,500,000` 演算程度

---

## R² の計算

返り値の `r_squared` は、Random Forest の**訓練データ上の決定係数**（OOB が計算困難な小サンプル時のフォールバック）:

```
R² = 1 − Σ_i (y_i − ŷ_i)² / Σ_i (y_i − ȳ)²
```

訓練データへの過学習があるため R² は高くなりがち。モデルの質を評価するには視覚的に曲面の滑らかさを確認することが望ましい。

---

## 計算量と性能目標

| N（試行数） | 学習時間の目安 | グリッド予測（50×50） |
| ----------- | -------------- | --------------------- |
| 50〜200     | < 100ms        | < 50ms                |
| 1,000       | < 500ms        | < 200ms               |
| 5,000       | < 2,000ms      | < 500ms               |

**目標: 2,000ms 以内**（UI のローディングスピナーで待機を許容）。

## 特性・限界

**強み:**

- 非線形・不連続な目的関数に対応可能
- 外れ値への頑健性が高い（多数決により影響が希薄化）
- 特徴量スケーリング不要（決定木は閾値比較のみ）
- 過学習を Bagging で自然に抑制

**弱み:**

- 訓練データの外挿が苦手（データ範囲外では定数値になる）
- Kriging と比べて予測の滑らかさが劣る（階段状の曲面になりやすい）
- 計算コストは Ridge より大きい（`O(N · T · D · n_grid²)` 相当）

---

## 使用場面の目安

```
目的関数の形が...

  線形に近い ──────────────────────────────────→ Ridge（高速）
  非線形・不連続・外れ値あり ──────────────────→ Random Forest
  滑らかな非線形（GP的な連続性を期待する） ────→ Kriging
```

- 試行数が多い（N ≥ 200）場合に効果的
- `R² > 0.7` なら曲面の傾向は信頼できる
- 曲面が粗い（ノイジー）場合は `n_trees` 増加または `max_depth` 削減を検討
