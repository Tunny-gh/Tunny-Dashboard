## 依存タスク

- **前提タスク**: [TASK-1604: clusterStore 実装 + テスト](TASK-1604.md)
- **後続タスク**: なし（最終タスク）

## 完了条件

- [ ] `LeftPanel.tsx` が `useClusterStore` をインポートして使用している
- [ ] `ClusterPanel` の `onRunClustering(space, k)` が `clusterStore.runClustering(space, k)` を呼び出す
- [ ] `clusterStore.isRunning` が `ClusterPanel` に渡される（プロップスが存在する場合）
- [ ] `clusterStore.elbowResult` が `ClusterPanel` に渡される（プロップスが存在する場合）

## 実装詳細

### 1. LeftPanel.tsx の ClusterPanel 接続更新 🔵

**信頼性**: 🔵 *REQ-VE-044 / dataflow.md フロー3a より*

実装ファイル: `frontend/src/components/panels/LeftPanel.tsx`

```tsx
import { useClusterStore } from '../../stores/clusterStore';

// LeftPanel コンポーネント内に追加
const { runClustering, estimateK, isRunning, elbowResult } = useClusterStore();

// ClusterPanel に渡す（実際の Props 型を確認して調整）
<ClusterPanel
  onRunClustering={runClustering}
  onEstimateK={estimateK}          // ClusterPanel が onEstimateK を持つ場合のみ
  isRunning={isRunning}            // ClusterPanel が isRunning を持つ場合のみ
  elbowResult={elbowResult}        // ClusterPanel が elbowResult を持つ場合のみ
/>
```

**注意**: `ClusterPanel` の実際の Props 型定義を `LeftPanel.tsx` 内または別ファイルで確認してから、存在するプロップスのみを渡すこと。

### 2. onEstimateK コールバックの対応 🟡

**信頼性**: 🟡 *ClusterPanel.tsx の実際の Props 型を確認して対応する（設計ヒアリング確認事項）*

実装時に `ClusterPanel` に `onEstimateK` プロップスが存在しない場合は追加しないこと。存在する場合は `clusterStore.estimateK` を渡す。
