## 実装詳細

### 1. sensitivity-heatmap ケース追加 🔵

**信頼性**: 🔵 *REQ-VE-050〜054 / dataflow.md フロー1より*

実装ファイル: `frontend/src/components/layout/FreeLayoutCanvas.tsx`

switch 文に追加するケースと、ラッパーコンポーネントの実装：

```tsx
// switch 文内
case 'sensitivity-heatmap':
  return <SensitivityHeatmapWrapper />;

// FreeLayoutCanvas.tsx 内部コンポーネント（フック使用のためラッパーが必要）
function SensitivityHeatmapWrapper() {
  const {
    sensitivityResult,
    isComputingSensitivity,
    sensitivityError,
    computeSensitivity,
  } = useAnalysisStore();

  useEffect(() => {
    if (!sensitivityResult && !isComputingSensitivity) {
      computeSensitivity();
    }
  }, []);

  // SensitivityWasmResult → SensitivityHeatmap が期待する形式へ変換
  const data = sensitivityResult
    ? {
        spearman: sensitivityResult.spearman,
        ridge: sensitivityResult.ridge,
        paramNames: sensitivityResult.paramNames,
        objectiveNames: sensitivityResult.objectiveNames,
      }
    : null;

  return (
    <SensitivityHeatmap
      data={data}
      isLoading={isComputingSensitivity}
      error={sensitivityError}
    />
  );
}
```

**注意**: switch 文の中でフックを直接呼び出せないため、ラッパーコンポーネントパターンを使用する。
