### 2. wasm_compute_sobol() の追加 🔵

**信頼性**: 🔵 *architecture.md Layer 1 出力JSON仕様・implementation-guide.md より*

**実装ファイル**: `rust_core/src/lib.rs`

```rust
#[wasm_bindgen(js_name = "computeSobol")]
pub fn wasm_compute_sobol(n_samples: u32) -> Result<JsValue, JsValue> {
    let start = web_sys::Performance::now_with_this(&js_sys::global()
        .unchecked_into::<web_sys::Window>()
        .performance()
        .ok_or_else(|| JsValue::from_str("no performance API"))?);

    let result = compute_sobol(n_samples as usize)
        .ok_or_else(|| JsValue::from_str("compute_sobol failed: insufficient data or no parameters"))?;

    let duration_ms = web_sys::Performance::now_with_this(...) - start;

    // JSON シリアライズ
    let json = serde_json::json!({
        "paramNames":      result.param_names,
        "objectiveNames":  result.objective_names,
        "firstOrder":      result.first_order,
        "totalEffect":     result.total_effect,
        "nSamples":        result.n_samples,
        "durationMs":      duration_ms
    });

    JsValue::from_str(&json.to_string()).map_err(|e| JsValue::from_str(&e.to_string()))
}
```

**注意**: `web_sys::Performance` の使用方法は既存 `wasm_compute_sensitivity` の実装パターンに合わせること。`serde_json` が既に Cargo.toml に含まれている場合はそのまま使用する。

### 3. WASM ビルド + wasm-bindgen 実行 🔵

**信頼性**: 🔵 *既存 WASM ビルドフロー・CLAUDE.md より*

```bash
# rust_core ディレクトリで実行

# wasm-bindgen でバインディング生成

  --out-dir ../frontend/src/wasm/pkg \
  --target web
```

**注意**: 既存のビルドスクリプト（`build.sh` や `package.json` の scripts）がある場合はそちらを使用すること。
