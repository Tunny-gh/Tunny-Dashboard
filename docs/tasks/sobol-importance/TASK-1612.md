### 1. SobolWasmResult インターフェースの追加 🔵

**信頼性**: 🔵 *interfaces.ts より*

**実装ファイル**: `frontend/src/wasm/wasmLoader.ts`

`SensitivityWasmResult` インターフェースの直後に追加:

```typescript
export interface SobolWasmResult {
  paramNames: string[]
  objectiveNames: string[]
  firstOrder: number[][]   // [paramIdx][objIdx]
  totalEffect: number[][]  // [paramIdx][objIdx]
  nSamples: number
  durationMs?: number
}
```

### 2. WasmLoader クラスへのフィールド追加 🔵

**信頼性**: 🔵 *architecture.md Layer 2・既存 WasmLoader 実装パターンより*

**実装ファイル**: `frontend/src/wasm/wasmLoader.ts`

既存の `computeSensitivity!:` フィールドの後に追加:

```typescript
computeSobol!: (nSamples: number) => SobolWasmResult
```

### 3. _initialize() への computeSobol バインド追加 🔵

**信頼性**: 🔵 *architecture.md Layer 2・implementation-guide.md より*

**実装ファイル**: `frontend/src/wasm/wasmLoader.ts`

`_initialize()` メソッド内に追加（既存の `computeSensitivity` バインドと同様のパターン）:

```typescript
this.computeSobol = (nSamples: number): SobolWasmResult => {
  const raw = wasmComputeSobol(nSamples)
  return JSON.parse(raw) as SobolWasmResult
}
```

**注意**: `wasmComputeSobol` は WASM pkg の import 名。既存の import 文に `computeSobol as wasmComputeSobol` を追加すること。import 元は `'./pkg/tunny_core'`。
