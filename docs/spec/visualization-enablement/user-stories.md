## エピック 1: 感度分析の可視化

### ストーリー 1.1: 感度ヒートマップで変数影響を一覧する 🔵

**信頼性**: 🔵 *tunny-dashboard-requirements.md REQ-090〜098 / TASK-802 memo より*

**私は** 解析担当者 **として**
**感度ヒートマップ（Spearman 相関 / Ridge β）を FreeLayoutCanvas に表示したい**
**そうすることで** どの変数がどの目的関数に効くか全体像を一覧できる

**関連要件**: REQ-VE-001, REQ-VE-011, REQ-VE-020, REQ-VE-030〜034, REQ-VE-050〜054

**詳細シナリオ**:
1. Journal ファイルを読み込み Study を選択する
2. Mode D（フリーレイアウト）に切り替え、右サイドバーから「Sensitivity」をドラッグしてキャンバスに追加する
3. チャートマウント時に自動で `computeSensitivity()` が呼ばれローディングスピナーが表示される
4. WASM 計算完了後、パラメータ（行）× 目的関数（列）のヒートマップが表示される
5. 上部の Spearman / Ridge β ボタンで指標を切り替えられる
6. 閾値スライダーで低感度パラメータを非表示にできる

**前提条件**:
- Journal ファイルが読み込まれており、完了試行数 ≥ 2
- パラメータ数 ≥ 1

**優先度**: Must Have

---

### ストーリー 1.2: Brushing 後に感度を再計算する 🔵

**信頼性**: 🔵 *tunny-dashboard-requirements.md REQ-098 より*

**私は** 解析担当者 **として**
**Pareto フロント上の解をブラシ選択した後、その部分集合での感度を再確認したい**
**そうすることで** 設計空間の特定領域における変数影響を局所的に把握できる**

**関連要件**: REQ-VE-002, REQ-VE-033

**詳細シナリオ**:
1. 感度ヒートマップを表示した状態で Pareto 散布図をブラシ選択する
2. ヒートマップが自動更新（または「再計算」ボタン押下）して選択サブセットの感度を表示する

**優先度**: Should Have

---

### ストーリー 1.3: パラメータ重要度を確認する 🔵

**信頼性**: 🔵 *tunny-dashboard-requirements.md REQ-090〜091 / ユーザヒアリングより*

**私は** 最適化研究者 **として**
**変数重要度（Spearman |ρ| または Ridge |β|）を降順ソートされたバーチャートで確認したい**
**そうすることで** どの変数が最も重要かを素早く判断できる**

**関連要件**: REQ-VE-060〜065

**詳細シナリオ**:
1. FreeLayoutCanvas に「Importance」チャートを追加する
2. チャートが表示され、パラメータ名を Y 軸・重要度スコアを X 軸とする水平バーが降順で表示される
3. 上部ドロップダウンで「Spearman |ρ|」と「Ridge |β|」を切り替えてバーが更新される

**前提条件**:
- パラメータ数 ≥ 1
- `analysisStore.sensitivityResult` が取得済み（未取得の場合はチャート表示時に自動計算）

**優先度**: Must Have

## エピック 2: クラスタリングの可視化

### ストーリー 2.1: クラスタリング結果を 2D 空間で確認する 🔵

**信頼性**: 🔵 *tunny-dashboard-requirements.md REQ-080〜087 / ユーザヒアリングより*

**私は** 設計エンジニア **として**
**クラスタリング実行後に全試行の 2D PCA 散布図をクラスタ色で確認したい**
**そうすることで** クラスタの分離状況と試行の分布を視覚的に把握できる**

**関連要件**: REQ-VE-003〜006, REQ-VE-040〜046, REQ-VE-070〜077

**詳細シナリオ**:
1. 左パネルの ClusterPanel で Target Space（param/objective/all）と k を設定して「Run」ボタンを押す
2. `clusterStore.runClustering(space, k)` が呼ばれ、PCA → k-means → 統計量の順に計算される
3. FreeLayoutCanvas に「Cluster View」チャートを追加すると、PCA 2D 座標にクラスタ色分けの散布図が表示される
4. 軸ラベル "PC1"・"PC2" と凡例「Cluster 0」〜「Cluster k-1」が表示される

**前提条件**:
- Journal ファイルが読み込まれており、完了試行数 ≥ k

**制約事項**:
- クラスタリング未実行時は EmptyState を表示

**優先度**: Must Have

---

### ストーリー 2.2: Elbow 法で最適なクラスタ数を推定する 🔵

**信頼性**: 🔵 *tunny-dashboard-requirements.md REQ-083 より*

**私は** 設計エンジニア **として**
**Elbow 法のグラフで推奨クラスタ数の候補を確認し、k を設定したい**
**そうすることで** データ主導で適切なクラスタ数を決定できる**

**関連要件**: REQ-VE-043, REQ-VE-005

**詳細シナリオ**:
1. ClusterPanel で Target Space を選択後、「Estimate k」ボタンを押す
2. `clusterStore.estimateK(space)` が呼ばれ、k=2〜10 の WCSS カーブが計算される
3. Elbow グラフと推奨 k（赤点・ラベル）が表示される
4. k 入力欄に推奨 k が自動入力（または手動で変更可能）される

**優先度**: Should Have

## エピック 3: 次元削減散布図の可視化

### ストーリー 3.1: PCA 2D 散布図で試行の分布を確認する 🔵

**信頼性**: 🔵 *tunny-dashboard-requirements.md REQ-082 / ユーザヒアリングより*

**私は** 最適化研究者 **として**
**全試行を PCA で 2 次元に投影した散布図を確認したい**
**そうすることで** 高次元データの全体的な分布とクラスタ構造を直感的に把握できる**

**関連要件**: REQ-VE-080〜087

**詳細シナリオ**:
1. FreeLayoutCanvas に「UMAP」チャートを追加する
2. チャートタイトル "Dimensionality Reduction (PCA)" と共に 2D 散布図が表示される
3. 点の色は現在のカラーモード（Pareto ランク・目的値・試行番号）に従う
4. 左上のセレクタで "PCA" が選択されている。"UMAP" は "Coming Soon" として disabled

**前提条件**:
- Journal ファイルが読み込まれており、完了試行数 ≥ 2
- `clusterStore.pcaProjections` が利用可能（ない場合は自動取得）

**優先度**: Must Have
