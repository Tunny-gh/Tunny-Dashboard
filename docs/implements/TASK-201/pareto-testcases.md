## 正常系テストケース

### TC-201-01: 2目的NDSortが既知のParetoフロントを正確に識別する
- **入力**: 4点 [(1,4),(2,3),(3,2),(4,1)] → 全てが非支配（正方形の頂点）
- **期待結果**: 全点 rank=1 🟢

### TC-201-02: 2目的で明確な支配関係がある場合
- **入力**: 3点 [(1,1),(2,2),(3,3)] → (1,1)が全てを支配
- **期待結果**: ranks=[1,2,3] 🟢

### TC-201-03: 4目的NDSortが正確に動作する
- **入力**: 3点 [(1,1,1,1),(2,2,2,2),(1,2,1,2)] → (1,1,1,1)が全てを支配、(1,2,1,2)と(2,2,2,2)の関係確認
- **期待結果**: ranks=[1,3,2]（(1,2,1,2)は(2,2,2,2)を支配） 🟢

### TC-201-04: 1目的の場合は全点Rank1
- **入力**: 5点 obj0=[3.0,1.0,4.0,1.5,2.0]
- **期待結果**: 全点 rank=1 🟢

### TC-201-05: maximize方向の目的が正確に処理される
- **入力**: 3点 obj0=[1,2,3]（maximize方向）→ 3が最大、支配
- **期待結果**: obj0=3 は rank=1、obj0=2 は rank=2、obj0=1 は rank=3 🟢

### TC-201-06: 2D Hypervolume が既知の値を返す
- **入力**: Pareto点 [(1.0,4.0),(2.0,2.0),(3.0,1.0)]、参照点 (5.0,5.0)
- **期待結果**: HV = (5-1)×(5-4) + (5-2)×(4-2) + (5-3)×(2-1) = 4 + 6 + 2 = 12.0 🟢

### TC-201-07: 1目的の場合 Hypervolume = None
- **入力**: 単目的 Study
- **期待結果**: `hypervolume: None` 🟢

### TC-201-08: Trade-off Navigatorが正確な順序を返す
- **入力**: 3点 [(1,4),(2,2),(4,1)]、weights=[0.5,0.5]、ideal=[1,1]
  - score0 = max(0.5×0, 0.5×3) = 1.5
  - score1 = max(0.5×1, 0.5×1) = 0.5
  - score2 = max(0.5×3, 0.5×0) = 1.5
- **期待結果**: score順 → インデックス1が最初（score=0.5） 🟢

### TC-201-09: Hypervolume推移が正確に計算される
- **入力**: 3試行（trial_id=[0,1,2]、obj0=[2.0,1.0,3.0]）、1目的はHV=None
- **期待結果**: hv_values は1目的なので全て0.0または None扱い 🟡
